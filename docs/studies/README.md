# Module & Recipe Studies

Each file here documents one component's lifecycle: the pre-work hypothesis (what we expected) and the post-mortem findings (what we observed).

This is the lab notebook for the framework. It answers: "what did we learn when we built X, and what does that tell us about the design space?"

## How to Add a Study

1. Copy `docs/study-template.md` to `docs/studies/<component_id>.md`
2. Fill **Part 1 (Hypothesis)** before writing any implementation code
3. Fill **Part 2 (Findings)** after the PR is merged
4. Reference the study in the PR description under **ADR Impact** if it surfaces generalizable rules
