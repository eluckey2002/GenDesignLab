02 / Philosophy

Charts are documents.

Most AI work is shown in charts that look the same as every other dashboard on the internet. Default Recharts. Default Plotly. Default everything.

Plumb's charts are different by intention. Hairline grids. Bronze data lines. Mono labels. Restraint. Numbers that overflow their frames when they earn it. Status colors only as state, never as decoration.

The point isn't decoration. It's that a Plumb chart should be readable from across the room and trustworthy from up close. The same way the brand reads.

A chart that requires a legend taller than itself probably wants to be a table.
03 / KPIs

Numbers that overflow their frame.

Display numerals in Fraunces, anchored by a hairline Bronze rule that runs past the card edge. The unit is a small italic Oxblood whisper, not a label. No border, no shadow — the number does the work.

The single number, oversized.
A single statistic given the full weight of the page. Fraunces 144pt. Unit set tiny in italic Oxblood. Hairline Bronze rule extends past the card's left edge to anchor without enclosing.

04 / Time Series

Hairline grids. Bronze lines.

Most defaults make the grid as loud as the data. Plumb does the opposite. The grid is barely there; the data is the only thing that earns weight.

04.1 / Single line
One series, traced.
One Bronze line over a hairline grid. A faint Bronze fill below the line gives shape without competing. Y-axis labels in mono on the left; x-axis on the bottom.
Use for a single metric over time — token cost, eval pass-rate, request volume.
Token cost · last 12 weeks
04.2 / Multi-line
Two or three series, in dialogue.
Bronze for the headline series. Slate-blue for the comparator. Sage for the threshold or target line. Never more than three. The legend is inline at each line's end, not a separate block.
Use for before/after over time, baseline vs. shipped, or model A vs. model B.
Eval pass-rate · v2 vs baseline
05 / Distribution

Where the long tail lives.

Latency distributions, response-time histograms, anything where the shape of the data matters more than the average. The default visualization treatment overweights the median; Plumb's makes the tail visible.

05.1 / Latency markers
P50, p95, p99 — at a glance.
A horizontal track with three Bronze markers showing where the percentiles land. Sage zone marks "in spec," Clay zone marks "out of spec" — both as low-opacity background fills. Mono labels above and below.
Use to summarize latency at a glance — in a status email, at the top of an SLO report, on a contract page.

06 / Comparison

Two states, one frame.

Before and after. A and B. Baseline and shipped. The point of comparison viz is to make the delta the headline — not to be even-handed about both sides.

06.1 / Before / After
Two bars, one delta.
Baseline bar in Ash (faded). Shipped bar in Bronze (full). The delta lives in the middle — Sage arrow if we improved, Clay if we regressed, with a Fraunces percentage. The bars are small; the change is the point.
Use in case studies. The single most-used comparison shape Plumb produces.

07 / Flow

Pipelines, drawn quietly.

Default flow charts use boxes and arrows. Plumb uses the Hairline Bridge primitive — diamond nodes connected by Bronze hairlines. No boxes. No arrows. The active node glows.

Diamond nodes mark each step. Bronze hairlines connect them. Active node glows. Done nodes filled. Failed nodes outlined in Clay. Step number, label, and duration stack vertically below each node.
Use for engagement phases, agent loops, build steps — any process you'd otherwise visualize with boxes and arrows.

A graph, drawn like a star chart.
Hairline Bronze paths connect nodes positioned on a non-linear grid. Active nodes glow. Past nodes are filled Bronze. Future nodes are hairline-only. Use for systems that don't move in a straight line.

09 / AI-Specific

Where the work actually lives.

Charts that exist because Plumb does AI work. Confusion matrices, eval-rate-over-time, token cost by model, citation grounding. The defaults are ugly — these are not.

Use for classifier evaluation. Always show both raw count and percent — one without the other lies.

11 / Implementation

Defaults, axes, restraint.

A small set of rules that make every chart in this library feel like part of one system. Hand off with the file.

Grid lines
Hairline color, 1px. Horizontal grids only — vertical grids appear only when categorical x-axis demands it. Never both.
Data lines
Bronze (primary), Slate Blue (comparator), Sage (target/threshold). 2px stroke. Never more than three series.
Axis labels
Geist Mono, 9px, Ash color, 0.1em letter-spacing, all caps for units. Never lower-case.
Annotations
Inline at the line's terminus, never in a separate legend. Mono, color-matched to the series.
Numerals
Fraunces variable, opsz 144 for display KPIs, opsz 36 for table cells. Tabular figures on (tnum) for alignment.
KPI rule
Hairline Bronze rule extends 56px past the card's left edge. Anchors without enclosing.
Status colors
Sage / Slate Blue / Plum / Oxblood / Clay only as state. Never as decoration. The legend uses the Status Ribbon's color set.
Active emphasis
Single radial Bronze glow on the most recent / active data point. filter: drop-shadow(0 0 6px Bronze). Never animated unless the chart is live.
Threshold lines
Bronze, 1px, dashed (3 4), 0.6 opacity. Inline label at the right end.
Sparklines
No axes. No labels. No grids. Just a Bronze line with optional 0.12 opacity Bronze fill below. 80×24px default.
Padding
Card surfaces use 40px horizontal, 32px vertical. Charts breathe inside that padding; never bleed to the edges except the KPI rule.
Legend rule
If the legend would be taller than the chart, the chart wants to be a table.
