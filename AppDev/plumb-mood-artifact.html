02 / Directions

Two takes on the mark.

Both directions are geometric, type-forward, and rooted in the brand's three meanings — vertical truth, depth, aplomb. One is literal. One is typographic. Pick the swing or pick the steady; we'll commit fully either way.

03 / Philosophy

Vertical truth as a design primitive.

The system is built from one idea, repeated thoughtfully. The plumb line — the oldest tool for measuring true vertical — appears across the brand as marks, dividers, callouts, status indicators, and document wayfinding.

Less decoration. More clarity. The constraint is the point.

What follows is a catalog of seven primitives. They are not ornament. Each one earns its place by doing a job — wayfinding, verification, hierarchy, status. Together they make Plumb's pages feel considered without being precious.

A brand that has ideas in the details earns trust faster than a brand that performs them.
04 / The Plumb Test

Four questions, run in order.

Before anything goes out — a deck, a model, a recommendation, a paragraph — it passes through these. Fail one, it goes back. Pass all four, ship it.

01 / Truth

Is it true?

We can back it with data — and we have. Sourced. Verifiable. If we're guessing, we say so. If we're hedging, we name what we don't know yet.

02 / Use

Is it useful?

It changes what you decide, not just what you read. If removing it doesn't change a single decision your team will make, it doesn't go in.

03 / Operability

Is it operable?

Someone else could run with this without us in the room. Hand-off ready. No tribal knowledge required, no "ask Evan" footnote.

04 / Clarity

Is it clear?

A first-time reader gets it without translation. If we needed to be there to explain, we didn't write it well enough. Try again.

If it doesn't pass all four, it isn't done.
05 / How We Work

Diagnose. Build. Hand off.

Three phases, each with its own discipline. The hand-off is the deliverable — if your team can't run it without us, we didn't finish the job.

Phase 01
Wk 1–2 · Diagnose

Read the room.

We start by understanding what you're actually solving. We read your data. We ask the uncomfortable questions because the comfortable ones rarely surface real constraints. Most of the value of an engagement is decided here, before anyone touches code.

Phase 02
Wk 3–8 · Build

Build for production.

We engineer for the real environment from day one — your messy data, your legacy systems, your latency budget. We document tradeoffs as we make them. By week six, your team is in the loop. By week eight, the integration is moving a number you already report on.

Phase 03
Wk 8–10 · Hand off

Step out of the room.

We're not load-bearing forever. We finish with your team owning the eval harness, the deployment, the on-call rotation. We stay close, but we step back. The deliverable is what works after we leave.

11 / Implementation

Notes for whoever picks this up.

Specs, defaults, and the quiet decisions that make the system feel like a system. Hand this off with the file.

Baseline grid
8 px. All vertical rhythm and component padding rounds to multiples.
Hairline weight
0.5px on Retina, 1px otherwise. Use Hairline color for borders; Bronze for marks.
Depth marker
Three equal divisions. Active third in Bronze; surrounding ticks in Hairline.
Tension line
Horizontal hairline starts at 20%, ends at 80%. Never margin-to-margin.
Bronze accent budget
No more than 3% of any composition. Bronze loses meaning at higher density.
Operational colors
Only as state. Never decorative. Sage = live; Clay = failed; Slate Blue = info; Amber = draft; Plum = experimental.
Fraunces axes
opsz 144 for display, opsz 14 for inline italics. Weights: 400 / 500 / 600.
Mono use
Status, metadata, terminal output. Never body. 0.12em letter-spacing on small caps.
Gradients · allowed
Value gradients within a single color family (Slate→Stone, Bronze→Bronze Dark). Soft surface gradients (≤15% value shift). Active-state radial glows. Hairline fades.
Gradients · banned
Hue gradients across families. Saturated rainbow gradients. Sunset/sky gradients. Anything decorative.
Gradient direction
Default top-to-bottom or top-center radial (gravity-aware). Diagonal (135° / 160°) only on full-bleed app cards.
Gradient test
A viewer should sense depth without consciously perceiving the gradient. If you can name it, it's too loud.
Oxblood discipline
Italic Fraunces only. Never on status, labels, or mono. Never adjacent to Clay in the same composition. Never naked — always lives inside prose or a callout.
Motion easing
cubic-bezier(0.4, 0, 0.2, 1). 200–400 ms. No bounce. Hairline reveals over fades.
Type stack
Fraunces (display) + Geist (body) + Geist Mono (mono). All three are open-source and freely available via Google Fonts. No commercial licensing required.
Mark construction
Direction A: tapered Bronze stroke + diamond bob at lower third. Direction B: bar weight = Fraunces 600's heaviest hairline.
